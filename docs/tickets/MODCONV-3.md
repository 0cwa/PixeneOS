# MODCONV-3 — Manifest format spec (secondary surface)

## Goal

Specify the manifest format the converter emits. Designed *after* the converter so the format is shaped by real conversion needs, not theoretical aesthetics.

## Acceptance

- `docs/planning/manifest-format.md` with: full schema, examples, signing model (per META-10), versioning policy, forwards-compat rules.
- Migration path from existing module formats to manifest format.
